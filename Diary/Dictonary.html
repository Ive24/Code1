Dictonary

This section of my learning diary is a collection of important terms and commands.

Git

Repository


The repository is a place where you can store your code and your files. It contains all information about your project, including a history of all the changes.

Commit


A commit is a save or permanent snapshot of your entire project/repository at acertain time. Like a "save point" in the project you can go back to. A commit should alwahs include a message to keep track of the changes. Git basically stores and manages commits.

Branch


A branch is a seperate version of the main repository. This allows you to work on multiple aspects of your project at the same time and test new things. You can imagine it as a tree where you working on different branches of the same tree (working tree).

Remote


Remote means that it's not the local version on your machine. A remote version of your repository is available for others (teammembers or the whole world). It allows you to work together with people all over the world. (remote, dt.: entfernt, abgelegen).

stage


Stage means to prepare a file finely for a commit. All modified files that will go into the next commit snapshot are stored in the staging are/index. Git allows you to commit only certain parts of the changes you have done since the last commit.

fetch


It's a command to download new data from a remote repository, but it does not integrate this data into the working files. It also does not modify, destroy or overwrite anything. It only updates the remote-tracking branches. This makes it useful to get a fresh view of what happend in a remote repository without affecting the local branch. It's important when working on a teamproject. You use fetch to connect your Git to the teammembers Gits and "collect" all of their new commits.

pull


Pull is used to fetch and integrate the changes from a remote repository to a local repository. With this command you can update your local repository with the latest changes from the remote repository. It can be seen as a combination of the commands pull and merge.

push


This command updates the remote branch with local commits. You update or publish your changes to the remote repository. It only updates the correspomding branch on the remote.

merge


It's used to merge changes from one branch into another or to incorporate changes from another repository. Merge integrates all changes of one or more branches in another one (dt.: Zusammenführen). It combines the history of the merged branches.

.gitignore


You can use a .gitignore file to specify which parts of your project should be ignored by Git. Git does not track files and folder specified in .gitignore. However all files that are already tracked by Git won't be ignored. It can be used for files that you don't want to share, like personal or temporary files.

rebase


This command allows you to reorganize your commit history and change the base of your branch.

squah


Squashing a commmit means to move the changes of one commit into it's parent so that you end up with one commit instead of two or more. This process can be reapeted multiple times. You can reduce n commit to a single one.

.git


The .git folder is a hidden folder that contains all the information that Git needs to manage your project's version control.

Sync Changes in VSCode


It will run a pull command followed by a push command.

HTML

HTML


HTML determines the structure and the content of web pages.
Hypertext: Text that is organized in order to connect related items and to link web pages (within a single website or between different ones).
Markup: Markup suggest the use of tags to define elements and add information. The markup includes special elements, which consists of the element name sourrounded by "<" and ">".
Language: a language that a computer system understands and uses to interpret commands

Syntax


Syntax refers to the rules and principles to write a correct code. Syntax is about to the structure.

Semantic


Semantic means that the tags convey (transfer, provide, dt.: vermitteln) the actual meaning of what they are used for. Semantic is about the meaning.

Tags


You have opening tags () and closing tags (). In between the two tags the content is written. Both tags and the content form an element. It is possible to put an element inside another element. Tags are used to define different types of content on a webpage. Different categories of tags:
- structure tags for the overall structure of the webpage
- image tags
- formatting tags to change font, color etc. of the shown text
- navigation tags to define connections with other resources
- input (form) tags to collect information and enable input
- audio and viedeo tags
- style and programming tags to define the program
- meta tags for non-visible information

CSS

CSS


CSS (Cascadind Style Sheets) is used to style your web content.

cascading


If you don't change the style it is going to cascade to the next section.

margin


How one box is placed to other boxes.

Selector


This is the HTML element name at the start of the ruleset. It defines the element(s) to be styled. To style a different element, change the selector.

Declaration


This is a single rule like color: red;. It specifies which of the element's properties you want to style.

Properties


These are ways in which you can style an HTML element. (In this example, color is a property of the

elements.) In CSS, you choose which properties you want to affect in the rule.

Property value


To the right of the property—after the colon—there is the property value. This chooses one out of many possible appearances for a given property. (For example, there are many color values in addition to red.)
